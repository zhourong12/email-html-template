# email-html-template 是干什么的？

你可以把它理解成：**给 AI 看的一份「写邮件说明书」**。  
装上以后，你在对话里让 AI 帮你写**正式发出去的那种网页邮件**（带排版、按钮、公司风格那种），它会按说明书来：先理清楚要写什么，再出成各大邮箱里不容易乱版的格式，还会多帮你**检查几遍**内容和版式。

适合：**活动通知、会员福利、产品更新、节日问候**这类要发一大群人的邮件；也适合公司内部**系统通知**类邮件（按你实际场景来）。

---

## 装上以后，AI 会多帮你做哪些事？

用大白话说：

1. **先问清楚**：给谁发、中文还是英文、有没有「亲爱的某某」这种要替换的名字、是不是广告类（广告类一般要留退订说明的位置）。  
2. **先把内容整理成一张「结构清单」**（不让 AI 一上来就糊一整页代码，减少乱码和错位）。  
3. **再按清单生成邮件网页版**，尽量在 **Gmail、阿里云发信、Outlook** 里都能正常看（Outlook 有时会略不一样，说明书里也会提醒）。  
4. **自动过几道检查**：链接安不安全、有没有不该出现的东西、手机和电脑关掉图片时还能不能看懂大意。  
5. **最后再出**：网页版 + 纯文字版，方便你复制到发信系统里。

邮件长得**好不好看**，说明书里也写了一些「别做成那种一眼像机器生成的丑模板」的规矩，都在同一个文件里，**不用**再装别的设计类说明书。

## 我不会技术，怎么装？

**最简单的办法：把下面这段话复制给你正在用的 AI 助手（比如 Cursor、Claude 里写代码的那个助手），让它帮你搞定。**

如果你只想发一句超短的，也可以试：

> `git@github.com:zhourong12/email-html-template.git`，帮我安装这个写邮件的 skills。

更稳妥、不容易装错的是下面这段完整说明：

> 请从 GitHub 下载这个仓库：`git@github.com:zhourong12/email-html-template.git`  
> 下载完成后，在仓库里找到一个叫 **`email-html-template`** 的文件夹（里面有一个 **`SKILL.md`** 文件）。  
> 请查一下我当前用的这款 AI 软件「官方说明里 skills 要放在哪个文件夹」，把这个文件夹按说明放好，**文件夹名字不要改**，还叫 `email-html-template`。

装好之后，你在对话里可以说：**「按邮件 HTML 那份 skill 来写」** 或 **「用 email-html-template」**，AI 就知道按这份说明书干活。

如果你更习惯用浏览器打开 GitHub，也可以复制这个链接下载压缩包：  
[https://github.com/zhourong12/email-html-template](https://github.com/zhourong12/email-html-template)

## 常见软件大概放在哪儿？（给愿意自己动手的人）

不同软件名字不一样，**以你软件自己的帮助文档为准**，这里只是常见说法：

- **Cursor**：项目里常有 `.cursor/skills` 一类文件夹。  
- **OpenCode**：常有 `.opencode/skills` 一类位置。  
- **Claude Code**：常有 `.claude/skills` 一类位置。  

你只要保证：**最后有一个文件夹叫 `email-html-template`，里面躺着 `SKILL.md`**，和软件要求对上就行。

本仓库里，这个文件夹目前放在 **`.cursor/skills/email-html-template/`** 下面——你从 GitHub 下载整个项目后，去这个路径里把它**整个文件夹**拷走即可。

## 发出去的邮件，主要想兼容哪些邮箱？

一般按 **Gmail、用阿里云发信、Outlook** 这类常见场景来写；**不是**绑定某一家发广告的平台，你用哪家发信工具，只要把生成的网页贴进去即可（具体按钮在你那家工具里）。

---

## 版本号在哪看？

打开 **`SKILL.md`** 文件最上面几行，里面有 **version**，那就是当前说明书版本。
